API Reference

Save Product

The Add Product API is a POST REST API, it accepts form-encoded request body(raw data), uses HTTP response codes, it requires authentication authorization(bearer token) and returns API response in JSON format.

Admin and Seller can add product into their inventory list, if you want to add product you should register as a Seller and login into AMP application.

Add product having 4 types of products such as Static, Digital, Digital/Static and Media types.

Once Seller added the product, those products will visible to Buyer account and Buyer can create campaigns with products and purchase.

Base URL
https://advertisingmarketplace.com:8080/api/save-product-details
Client libraries
PHP
LARAVEL

Authentication

API requires Authorization token and we need authentication keys as username and password to generate authorization token.

Authorization token is a secret key and it expires in every one hour.

If API request includes expired key, response returns an authentication error. Then we need to login again with authentication keys and generate new authorization token.

Errors

AMP uses conventional HTTP response codes to indicate the success or failure of an API request. In general: Codes in the 2xx range indicate success. Codes in the 4xx range indicate an error that failed given the information provided (e.g., a required parameter was omitted, a charge failed, etc.)

Some 4xx errors that could be handled programmatically (e.g., a card is declined) include an error code that briefly explains the error reported.

HTTP status codes
200 - OK Getting response successfully as expected
400 - Bad Request Required parameters are missed
401 - Unauthorized No valid API authentication key
402 - Request Failed Getting valid parameters but response failed
403 - Forbidden API key doesn't have permissions to execute the request
500 - Server Errors Something went wrong in API
